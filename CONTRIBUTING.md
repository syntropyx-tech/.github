# Contributing

## Issues

Use the appropriate issue template. Fill out all required fields.

## Pull Requests

1. Create a branch from `main` (sometimes `master` in older repos)
2. Make changes
3. Ensure tests pass
4. Open PR using the template
5. Request review

## Commit Messages

Please use conventional commits:

- `feat:` new feature
- `fix:` bug fix
- `refactor:` code change without behavior change
- `chore:` maintenance
- `docs:` documentation

## Code Review

All PRs require at least one approval before merge.
